# Spring Boot API Review Skill

## 什么时候使用

当任务涉及新增或修改 Spring Boot HTTP 接口、Controller、DTO、接口兼容性或接口测试时使用。

## 输入材料

- 本次接口变更的 Controller / DTO / Service 代码。
- 关联的错误码、权限规则、数据库字段或下游服务说明。
- 已有测试或调用方依赖。

## 执行流程

1. 先确认接口契约：路径、HTTP 方法、请求 DTO、响应 DTO、字段必填规则是否清楚。
2. 再检查兼容性：是否删除字段、改变字段含义、改变状态码或破坏已有调用方。
3. 检查业务边界：租户、用户权限、幂等、状态流转、参数校验是否落在服务层。
4. 检查异常处理：业务异常是否有明确错误码，是否避免把内部异常直接暴露给调用方。
5. 检查测试：正常路径、参数非法、权限不足、重复提交、边界状态至少要覆盖关键样本。

## 输出要求

输出时先列风险，再列修改建议。不要只给泛泛的“代码结构清晰”之类评价。
